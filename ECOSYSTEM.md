# The keys7 ecosystem — a machine ↔ music loop

keys7 is the hub of a small system that connects a physical piano, a DAW
(Renoise), audio analysis and music notation around a couple of shared pivot
formats. This document explains the pieces and how they fit; each tool has its
own README for detail.

## The tools

### In this repo (Go)

- **keys7** — live harmony assistant on a MIDI piano (developed on a Yamaha
  P-125). A TUI reads chords / key / mode in real time, journals the session as
  JSONL, and lets the player hand turns to an assistant with cue gestures.
- **play7** — the output twin: plays a play7 JSON sequence **or a `.mid`** on a
  MIDI device, with *playing styles* (timing/velocity humanisation, chord roll,
  legato/staccato, auto-pedal). `--export` writes the sequence back to a `.mid`.
- **export7** — turns a session journal into a `.mid` (the bridge from a live
  take to notation).
- **internal/smf** — a pure-Go Standard MIDI File reader/writer, the `.mid`
  lingua franca shared by play7 and export7.

### Separate repos (Marc's)

- **xrns7** (Go) — reads Renoise `.xrns` files headlessly into notes JSON or
  play7 sequences, with a melodic-reduction filter (`--no-drums` / `--drop` /
  `--keep`) and a per-track classifier (drum / fx / melodic).
- **score7** (Python) — audio analysis: key, chords, melody, tempo/meter, stem
  separation. The view from the *rendered sound*, where the others work from
  symbolic data.

### Third-party links (not Marc's dev)

- **ReMCP** — drives Renoise over MCP: an assistant builds and applies motifs
  into patterns. The **write** side into the DAW (xrns7 is the **read** side).
- **MuseScore 4** — engraves a `.mid` into classical notation (PNG / PDF /
  MusicXML). The notation renderer downstream of `play7 --export` / `export7`.

## Pivot formats

- **play7 sequence JSON** — `{ "tempo", "voices": [ { "steps": … } ] }`. The
  lingua franca for *symbolic playback*: xrns7 emits it, play7 consumes it.
- **`.mid` (via internal/smf)** — the lingua franca for *interchange*:
  play7 ⇄ MuseScore, export7 → MuseScore.
- **session journal JSONL** — keys7's faithful capture of a live take
  (note/pedal/chord/key/melody/cue events); export7 turns it into a `.mid`.

## The chain

```
   ReMCP ──writes──▶ Renoise ──.xrns──▶ xrns7 ──play7 JSON──┐
                                       (melodic filter)      │
                                                             ▼
 you @ piano ──MIDI──▶ keys7 ──journal──▶ export7 ──.mid──┐  play7 ──styles──▶ P-125 (sound)
                                                          │  │   ▲
                                                          ▼  ▼   └── reads .mid back
                                                       MuseScore ◀── play7 --export
                                                       (score: PNG/PDF/MusicXML)

 rendered audio ─────────────────────────────▶ score7 ──▶ key / chords / melody
```

## How they complement (and don't overlap)

Each capability has exactly **one home** — that is what keeps the system free of
redundancy:

| Capability | Lives in |
|---|---|
| Renoise `.xrns` → notes, melodic reduction | xrns7 |
| Playback, styles, pedal | play7 |
| Live symbolic theory (chords, modes, degrees, cadences) | keys7 / `internal/theory` |
| Audio → key / chords / melody | score7 |
| `.mid` read/write | `internal/smf` |
| Live capture (journal) | keys7 / `internal/session` |

Two deliberate complements:

- **xrns7 reads the project file; score7 reads the rendered sound.** score7
  recovers exactly what xrns7 cannot: notes generated by **VST-internal
  arpeggiators** are absent from the `.xrns` (the pattern only holds the trigger
  note) but present in the audio.
- **ReMCP writes into Renoise; xrns7 reads out of it** — the two directions of
  the DAW bridge.

## Roadmap

### Near term — close today's gaps

1. **play7 arpeggiator** — re-arpeggiate a held chord (`up`/`down`/`updown`,
   rate, octave span). The #1 gap: a piano reduction loses the arpeggiated lines
   that live in VST arps, absent from the `.xrns`.
2. **xrns7 `--no-fx`** — drop trigger/effect tracks (the `fx?` hint already
   shows in `info`); completes the melodic sieve.
3. **SMF type-1 multi-track export** — one MIDI track per voice, so MuseScore
   lays melody / chords / bass on separate staves.

### Mid term — consistency & complement

4. **Rename `xrns7 play` → `seq`** — it emits a sequence, it doesn't play; the
   one naming inconsistency in the system.
5. **Arp bridge via score7** — transcribe the rendered audio to recover what
   xrns7 can't read (VST-internal arpeggiators), realising the xrns7 ↔ score7
   complement.

### Long term — vision

6. **keys7 real-time over the mesh** (SSE/MCP, "like OBS") instead of the file
   pull — the v2 leap, with style-aware coaching from flux7-memory and the
   Renoise analyses corpus.
7. **Packaging / tagged releases** across the tools.

### Standing note (not a TODO)

Harmonic theory lives in both keys7 (Go, symbolic) and score7 (Python, audio) —
unavoidable across the language/domain split; there is no single source of truth
for chord conventions.
